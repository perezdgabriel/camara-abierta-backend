## Phase 2: Data Collection Stabilization

### 2.1 Bill ingestion flow

Current flow (already implemented):

1. `OpenDataCamaraClient.get_mensajes_x_anno()` + `get_mociones_x_anno()` → list of bulletins
2. `SenadoAsync.fetch_bills_parallel()` → bill detail from Senado
3. `BillParser.parse_bill()` → normalized payload
4. `sync_bill` task → `upsert_bill()` → DB
5. `index_bill` task → Elasticsearch (deferred — skip for v0.1)
6. Voting sessions dispatched from bill data

What needs fixing:

- Test end-to-end flow with real APIs
- Verify `BillParser` correctly maps Senado fields to model
- Ensure `IngestorState` tracking works for incremental syncs

### 2.2 Legislator ingestion flow

Current flow:

1. `SenadoClient.get_senadores_vigentes()` → senators
2. `OpenDataCamaraClient.get_diputados_periodo_actual()` → deputies
3. `LegislatorParser` → normalized payload
4. `sync_legislator` task → `upsert_legislator()` → DB

What needs fixing:

- Test both sources end-to-end
- Verify party matching works
- Verify district/circumscription assignment

### 2.3 Voting session ingestion flow

Current flow:

1. `SenadoClient.get_bills_by_date()` → bulletins modified since date
2. `SenadoClient.get_votes_by_bulletin()` → votes per bulletin
3. `VoteParser.parse_senate_vote()` → normalized payload
4. `sync_voting_session` task → `upsert_voting_session()` → DB

What needs fixing:

- Test the `since` date incremental flow
- Verify individual vote matching to legislators

## Phase 4: AI Summaries for Bills

### 4.1 Create bill summary LLM function

- Add `generate_bill_summary(full_text: str) -> str` to `app/services/llm.py`
- Prompt: plain-language Spanish, 2-3 paragraphs (what it proposes, who it affects, why it matters)
- Use Gemini first, OpenWebUI as fallback (existing pattern)

### 4.2 Create Celery task

- Add `generate_bill_ai_summary` task to `app/tasks/bills.py`
- Triggered automatically after `sync_bill` completes
- Reads `bill.full_text`, calls LLM, writes `bill.ai_summary` + `bill.ai_summary_updated_at`

### 4.3 Handle missing full text

- `index_bill` task already extracts PDF text if `full_text` is None and `full_text_url` exists
- AI summary task should wait for `full_text` to be available
- If no `full_text` and no `full_text_url`, skip AI summary
